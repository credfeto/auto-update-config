# Repository Structure Instructions

This repository manages configuration files that are synchronised across multiple GitHub repositories.

## Purpose

Changes made here are propagated to downstream repos (funfair and personal). When a file needs to be added to, removed from, or updated in multiple repos, the relevant `templates.json` is the correct place to record that change.

## Key Config Files

### `funfair/templates.json` and `personal/templates.json`

These are the primary files to edit. Both follow the same schema but cover different sets of repos. **When adding or removing files, update both unless the change is intentionally scope-limited.**

#### Schema sections

| Section | Purpose |
|---|---|
| `general.files` | Files copied verbatim to every managed repo. Key = relative path in repo, value = human description. |
| `general.mirror-folders` | Entire folders mirrored to every managed repo. Key = folder path, value = description. |
| `general.partial-files` | Files where only a delimited portion is synchronised (e.g. the globally-maintained header of `ai/local/index.md`). |
| `dotNet` | .NET-specific settings: `global-json`, `resharper-dotsettings`, and per-file entries. |
| `gitHub` | GitHub-specific settings: dependabot generation, label generation, actions, linters, and extra files. |
| `cleanup.files` | Files that should be **deleted** from managed repos. Key = path, value = reason (e.g. "Obsolete action", "Replaced by X"). |

#### Common operations

- **Adding a new shared file** → add an entry to `general.files` in both `funfair/templates.json` and `personal/templates.json`.
- **Mirroring a new folder** → add an entry to `general.mirror-folders` (personal only currently uses this for `.github/instructions` and `ai/global`).
- **Removing an obsolete file from all repos** → add an entry to `cleanup.files` in both templates with a clear reason string.
- **Removing a GitHub Actions workflow** → add to `cleanup.files` with a reason explaining what replaced it.

### Repo Lists

| File | Content |
|---|---|
| `personal/repos.lst` | SSH URLs of personal repos to manage. Auto-generated by `build_personal.py`. |
| `funfair/repos.lst` | SSH URLs of funfair repos to manage. |
| `funfair/funfair-wrecking-league.lst` | Wrecking league repo list. |
| `funfair/repos-casino.lst` | Casino repo list. |
| `funfair/repos-eth.lst` | Ethereum repo list. |
| `funfair/repos-wallet.lst` | Wallet repo list. |

### Scripts

| Script | Purpose |
|---|---|
| `build_personal.py` | Fetches all non-archived, non-fork personal repos from GitHub and writes `personal/repos.lst`. |
| `update_repos.py` | Updates GitHub branch protection settings for repos in `personal/repos.lst`. |

## Root Config Files

The root-level config files (`.editorconfig`, `.globalconfig`, `.csharpierrc`, `.gitignore`, `.gitattributes`, `.sqlfluff`, `.tsqllintrc`, `.gitleaks.toml`) are the **source of truth** — they are the files that get synchronised out to managed repos via `general.files`.
